# Типовые ошибки и их решения в проекте Sputnik Studio v8

В процессе разработки мы столкнулись с рядом повторяющихся проблем. Этот файл фиксирует их и способы решения, чтобы в будущем не тратить время на повторный поиск.

---

## 1. Ошибки сборки (GitHub Actions)

### 1.1. `Dependencies lock file is not found`
**Ошибка:**  
npm error code ENOENT
npm error Could not read package.json

text
**Причина:**  
GitHub Actions использует `npm ci`, который требует точного `package-lock.json`. Если файл отсутствует или не совпадает с `package.json`, сборка падает.

**Решение:**  
- Убедиться, что `package-lock.json` закоммичен.  
- Или в `.github/workflows/deploy.yml` заменить `npm ci` на `npm install` (но это замедляет сборку).  
- Лучше всегда коммитить `package-lock.json`.

---

### 1.2. `Cannot find module` при импорте
**Ошибка:**  
Error: Cannot find module '../features/...'

text
**Причина:**  
- Файл физически отсутствует в репозитории.  
- Опечатка в имени файла или пути (регистр важен).  
- Импортируемый компонент не экспортирован (отсутствует `export`).

**Решение:**  
- Проверить существование файла по указанному пути.  
- Убедиться, что имя файла и его экспорт совпадают с импортом.  
- В случае переименования — обновить все импорты.

---

### 1.3. `TS6133: 'variable' is declared but its value is never read`
**Ошибка:**  
TypeScript ругается на неиспользуемые переменные.

**Причина:**  
- В коде остались импорты, которые не используются.  
- Объявлены переменные, которые не применяются.

**Решение:**  
- Удалить неиспользуемые импорты и переменные.  
- Если переменная нужна для будущего, можно добавить `eslint-disable-next-line` или временно закомментировать, но лучше убрать.

---

### 1.4. `TS6133: 'property' is declared but its value is never read` в JSX
**Ошибка:**  
При рендере списка используется `key`, но переменная `index` не используется.

**Решение:**  
- Если `index` не нужен, можно заменить на `(_, idx)` или использовать уникальный `id`.  
- Либо использовать `index` в компоненте (например, для отладки).

---

## 2. Проблемы с роутингом (HashRouter)

### 2.1. Прямой переход на `/calculations` даёт 404
**Ошибка:**  
GitHub Pages возвращает 404 при переходе на подстраницу, потому что сервер ищет файл `calculations.html`.

**Решение:**  
- Использовать `HashRouter` вместо `BrowserRouter`.  
- В `App.tsx` заменить `BrowserRouter` на `HashRouter`, удалить `basename`.  
- После этого все ссылки будут иметь вид `/#/dashboard`, `/#/projects` и т.д., и прямые переходы работают.

---

### 2.2. Параметры URL не читаются в `ProjectsPage` при `HashRouter`
**Ошибка:**  
`useSearchParams` не видит параметры после `#`.

**Решение:**  
- Использовать `useLocation` и парсить `location.hash` вручную.  
- Пример:
  ```ts
  const hash = location.hash;
  const queryStart = hash.indexOf('?');
  const query = hash.slice(queryStart + 1);
  const params = new URLSearchParams(query);
  const id = params.get('id');
3. Проблемы с модалками
3.1. Модалка не закрывается или не открывается
Причина:

В global.css у .modal display: none по умолчанию, а при открытии нужно добавить style={{ display: 'flex' }}.

Неправильно передаётся состояние isOpen.

Решение:

В компоненте модалки проверять isOpen и рендерить только если true.

Убедиться, что при закрытии вызывается onClose и родитель меняет состояние.

3.2. Модалка имеет неправильные стили (прозрачная, огромная)
Причина:
Глобальные стили .modal-content задают background: var(--bg-panel), который может быть полупрозрачным, а также min-width: 500px.

Решение:

Переопределить стили для конкретной модалки с помощью уникального класса и !important или инлайн-стилей.

В демо-примере модалка была компактной, непрозрачной – её стили можно взять за основу.

4. Проблемы с трактами
4.1. Устройства не добавляются или не отображаются
Причина:

В tractsSlice.ts в addDeviceToTract не указана колонка (требуется column).

Короткое имя генерируется неправильно (не учитываются все устройства).

Решение:

Проверить, что при вызове addDeviceToTract передаётся column ('source' | 'matrix' | 'sink').

В ActiveTract.tsx в handleAddDevice вычислять префикс и номер на основе всех устройств в тракте.

4.2. Редактирование устройства не сохраняется
Причина:

Не вызывается updateDeviceInTract или обновление не доходит до слайса.

В DeviceEditModal не передаётся onSave.

Решение:

В ActiveTract создать функцию handleUpdateDevice, которая диспатчит updateDeviceInTract.

Передать её в DeviceEditModal как onSave.

В модалке вызывать onSave(editedDevice).

4.3. Статистика портов и PoE не обновляется
Причина:

В recalcTract не учитываются порты и PoE.

Не вызывается пересчёт после изменений.

Решение:

В recalcTract суммировать totalPorts и usedPorts из всех коммутаторов, usedPoE из устройств с poeEnabled.

Вызывать recalcTract в useEffect после каждого изменения устройств или настроек видео.

5. Проблемы с каруселью
5.1. Кнопки навигации не появляются
Причина:

Embla Carousel требует передать navigation в useEmblaCarousel.

Кнопки должны быть вне внутреннего контейнера, но привязаны к emblaApi.

Решение:

В ProjectsCarousel добавить emblaApi и обработчики scrollPrev/scrollNext.

Кнопки разместить за пределами контейнера карусели (относительное позиционирование).

5.2. Карточки в карусели разной высоты
Причина:

Содержимое карточек может быть разным (например, разное количество строк в метаданных).

Решение:

В ProjectCard задать фиксированную высоту или использовать flex-растяжение.

В CSS .project-card добавить height: 100% и display: flex; flex-direction: column;.

6. Стили и адаптивность
6.1. Иконки в компактном сайдбаре не центрируются
Причина:

В Sidebar.css для .sidebar.collapsed .section-header задано justify-content: center, но иконка остаётся инлайн-элементом.

Решение:

Добавить display: flex; justify-content: center; для .sidebar.collapsed .section-header i:first-child.

Убрать margin-right.

6.2. Фокус на аккордеонах пропадает в развёрнутом виде
Причина:

В global.css есть правило button:focus, которое может переопределять.

Решение:

Использовать более специфичный селектор .section-header:hover и .section-header:focus.

Добавить outline: none для элементов, которые не должны иметь стандартный фокус.

7. Ошибки при работе с Redux
7.1. store не обновляется после dispatch
Причина:

Слайс не подключён в store/index.ts.

Используется не тот экшен.

Решение:

Проверить, что слайс импортирован и добавлен в reducer.

В компоненте использовать useDispatch и импортировать экшен из правильного слайса.

7.2. useSelector возвращает старые данные
Причина:

Изменение состояния не вызывает перерендер (мутация объекта).

В слайсе используется state.push без возврата нового объекта.

Решение:

Всегда использовать immer (Redux Toolkit делает это автоматически).

Не мутировать state напрямую без immer (но с Toolkit это разрешено, однако если выйти за пределы, могут быть проблемы).

Убедиться, что редьюсер возвращает новый объект или мутирует правильно.

8. Ошибки при экспорте/импорте JSON
8.1. После импорта проекты не появляются
Причина:

Данные импортируются в IndexedDB, но Redux не обновляется.

В storage.ts после db.projects.bulkAdd не вызывается dispatch(setProjects(...)).

Решение:

После загрузки в базу обновить Redux:

ts
const projects = await db.projects.toArray();
dispatch(setProjects(projects));
9. Ошибки при работе с Git и GitHub
9.1. После коммита сборка не запускается
Причина:

Ветка не main (Actions может быть настроен только на main).

Файл .github/workflows/deploy.yml отсутствует или имеет ошибки.

Решение:

Проверить, что файл существует и синтаксически корректен.

Убедиться, что в настройках репозитория Actions разрешены.

9.2. Сборка падает из-за неправильного base в vite.config.ts
Ошибка:
Страница пустая, в консоли 404 на main.js.

Причина:
base не совпадает с именем репозитория (например, base: '/sputnik-studio-react/').

Решение:

Установить base: '/sputnik-studio-react/' (для репозитория rawcam/sputnik-studio-react).

В package.json добавить "homepage": "https://rawcam.github.io/sputnik-studio-react".
