## 2. Реализованный функционал

### 2.1. Роутинг и страницы

- Используется **HashRouter** (для корректной работы на GitHub Pages).
- Маршруты:
  - `/` → перенаправление на `/dashboard`
  - `/dashboard` – главная панель управления
  - `/projects` – управление проектами
  - `/calculations` – калькуляторы и тракты (сайдбар появляется только здесь)
  - `/templates` – конструктор документов (заглушка)

### 2.2. Топбар

- Логотип, навигация (4 кнопки), переключатель темы (светлая/тёмная), кнопки «Сохранить» и «Экспорт» (пока заглушки).
- **Переключение ролей** – выпадающий список справа (директор, ГИП, инженер, проектировщик, логист). Роль хранится в `localStorage` и влияет на видимость виджетов и кнопок.

### 2.3. Сайдбар

- Показывается **только** на странице `/calculations`.
- Состоит из аккордеонов, конфигурация в `sidebarModules.ts`.
- Аккордеоны можно сворачивать/разворачивать, состояние сохраняется в `localStorage`.
- **Сортировка аккордеонов** (drag-and-drop) реализована через `@dnd-kit/sortable`. Порядок сохраняется в `localStorage`.
- **Компактный режим** – кнопка в хедере сайдбара сворачивает его до 80px, скрывая текст и оставляя только иконки.
- Стили иконок, отступы, фокус – исправлены.

### 2.4. Дашборд (DashboardPage)

- Сетка из 5 колонок (grid).
- **Виджеты**:
  1. **Финансы компании** (только для директора): общая выручка, маржа, рентабельность, кассовый разрыв. Данные из `useFinance`.
  2. **Финансы проектов** (директор/ГИП): топ-3 проектов по марже, проекты с отрицательной маржой.
  3. **Сервис и регламент** – ближайшие выезды (из `serviceVisits`).
  4. **Загрузка сотрудников** – инженеры и РП, загрузка в процентах (может быть >100%).
  5. **Риски и уведомления** – финансовые и технические риски (превышение бюджета, высокая загрузка сети).
- **Карусель активных проектов** – внизу дашборда. Использует `embla-carousel-react`. Кнопки навигации вынесены за пределы контейнера (на десктопе), на мобильных скрыты. При клике на карточку происходит переход на страницу `/projects?id=...` и открываются детали проекта. Карточки имеют единую высоту, приоритетные проекты выделены обводкой.

### 2.5. Проекты (ProjectsPage)

- Список проектов в виде карточек (компонент `ProjectList`).
- Создание нового проекта (модальное окно `CreateProjectModal`) – доступно для ролей директор/ГИП.
- При клике на карточку (или переходе из карусели) открывается детальная страница проекта (`ProjectDetail`) с вкладками:
  - **Общая информация** – редактируемые поля: название, категория, статус, бюджет, инженер, РП, дата, прогресс, приоритет.
  - **Финансы** – доходы и расходы (динамические списки), плановая/фактическая маржа, рентабельность, график денежных потоков.
  - **Сервис** – список сервисных выездов (добавление, редактирование, удаление).
  - **Дорожная карта** – план/факт по этапам (таблица с датами).
- Сохранение изменений обновляет Redux и IndexedDB.
- Демо-данные (3 проекта) создаются при первом запуске.

### 2.6. Калькуляторы и тракты (CalculationsPage)

- **Страница расчётов** имеет две области: сайдбар (настройки) и основную область (активный тракт или калькулятор).
- **Сайдбар** содержит:
  - VIDEO – настройки разрешения, субдискретизации, FPS, цветового пространства, битности; расчёт битрейта.
  - NETWORK – среда передачи, multicast, QoS, тип сети, синхронизация, резервирование.
  - LED, SOUND, VC, Эргономика ЭКП, ЭНЕРГИЯ – кнопки вызова калькуляторов на рабочий стол.
  - TRACTS – список трактов, кнопки «Новый тракт» и «Отобразить все тракты».
  - СЕТЬ – статистика (битрейт, загрузка, порты, multicast, QoS).
  - ПИТАНИЕ – статистика (PoE, мощность, тепло).
  - УПРАВЛЕНИЕ – экспорт/импорт JSON, сохранение/загрузка, сброс проекта, печать, Wiki.
- **Активный тракт** (`ActiveTract`) – отображается в основной области, когда выбран тракт.
  - Три колонки: «Начало тракта», «Коммутация», «Конец тракта».
  - В каждой колонке список устройств, кнопка «+ Добавить устройство».
  - Карточка устройства показывает модель, короткое имя, мощность, PoE, USB, Ethernet, задержку.
  - При клике на карточку открывается модалка редактирования (`DeviceEditModal`), где можно изменить мощность, PoE, USB, Ethernet.
  - При изменении Ethernet/PoE устройство автоматически подключается к первому доступному коммутатору (если есть) – **логика нестабильна** (порты не занимаются, PoE не учитывается).
- **Калькуляторы на рабочем столе**:
  - LED, звук, ВКС, эргономика, питание – открываются по клику на соответствующие кнопки в сайдбаре.
  - Содержат поля ввода, результаты и кнопку «Добавить в тракт» (добавляет устройство в активный тракт).
- **Все тракты** (`AllTractsView`) – отображается по кнопке «Отобразить все тракты», показывает список трактов с суммарной задержкой, битрейтом, мощностью.

### 2.7. Хранилище данных

- **IndexedDB** (Dexie) – таблицы `projects`, `companyExpenses`, `serviceVisits`.
- **Redux** – дублирует данные для быстрого доступа и реактивности.
- **Экспорт/импорт JSON** – работает (кнопки в разделе «УПРАВЛЕНИЕ»).
- **Сброс проекта** – очищает базу и создаёт демо-данные.

### 2.8. Роли и авторизация

- Роли: `director`, `pm`, `engineer`, `designer`, `logist`.
- Авторизация локальная (хранится в `localStorage`). Переключение ролей в топбаре.
- В зависимости от роли скрываются/показываются виджеты и кнопки (например, финансовые виджеты – только для директора/ГИП).

### 2.9. Стили

- CSS-переменные для светлой и тёмной темы.
- Стеклянные карточки, скруглённые углы, плавные переходы.
- Адаптивность под мобильные устройства (медиазапросы).

## 3. Что не работает (баги и недочёты)

### 3.1. Тракты

- **Порты коммутаторов не занимаются** – при включении Ethernet устройство не резервирует порт, статистика «СЕТЬ» показывает 0 занятых портов.
- **PoE-бюджет не учитывается** – при включении PoE не вычитается мощность из бюджета коммутатора.
- **Задержка в кадрах** – в `ActiveTract` добавлено отображение, но расчёт может быть неточным.
- **Автоматическое подключение к коммутатору** – часто выдаёт ошибку «Нет подходящего коммутатора», даже если он есть.
- **Статистика сайдбара (СЕТЬ, ПИТАНИЕ)** – не всегда обновляется после изменений в тракте.
- **Редактирование устройств** – изменения мощности, PoE, USB, Ethernet применяются, но без учёта портов и PoE.

### 3.2. Проекты

- **Дорожная карта** – добавлена, но тестировалась слабо.
- **Финансовые расчёты** – маржа, рентабельность, кассовый разрыв работают, но требуется дополнительное тестирование.

### 3.3. Карусель

- Кнопки навигации на мобильных скрыты (по замыслу), свайп работает.
- При клике на карточку происходит переход на страницу проектов с параметром `id`, детали открываются – **работает** (после последних исправлений).

### 3.4. Калькуляторы

- LED, звук, ВКС, эргономика, питание – имеют базовую логику, но могут быть неточности в расчётах (не критично для демо).

### 3.5. Роли

- Переключение ролей влияет на видимость, но некоторые элементы (например, кнопка «Новый проект») могут быть доступны не тем ролям.

### 3.6. Стили

- Небольшие недочёты: отступы, размеры шрифтов, адаптивность на планшетах.
- В компактном режиме сайдбара иконки центрируются не идеально.

- **Интеграция с внешними API** (цены, наличие) – не требуется.

## 5. Логика переключения разделов и работы карусели

### 5.1. Переключение между страницами

- Используется `HashRouter`. При клике на кнопки в топбаре меняется хеш (например, `#/projects`).
- Компонент `App` рендерит `Sidebar` и `main` с `Routes`.
- Страница `/calculations` – сайдбар показывается (условие в `Sidebar.tsx`).

### 5.2. Карусель активных проектов

- Компонент `ProjectsCarousel` получает массив проектов и `onSelectProject`.
- Использует `useEmblaCarousel` с опциями: `loop: false`, `align: 'start'`, `slidesToScroll: 1`, адаптивные breakpoints.
- Кнопки навигации (`prev`/`next`) управляют `emblaApi`.
- Контейнер карусели имеет `max-width: 1400px`, кнопки вынесены за его пределы с помощью отрицательных отступов (в CSS).
- При клике на карточку вызывается `onSelectProject`, который в `DashboardPage` выполняет `navigate('/projects?id=' + project.id)`.
- На странице проектов параметр `id` извлекается из `location.hash` (через `URLSearchParams`), и открывается детальный вид.

### 5.3. Переход из карусели в проект

- **Механизм**:
  1. Пользователь кликает на карточку в карусели.
  2. Вызывается `handleSelectProject(project)` в `DashboardPage`.
  3. `navigate('/projects?id=${project.id}')` изменяет URL на `#/projects?id=...`.
  4. `ProjectsPage` в `useEffect` следит за `location.hash`, извлекает `id` и находит проект в `projects`.
  5. Если проект найден, рендерится `ProjectDetail` с этим проектом.
  6. При нажатии кнопки «Назад» в деталях выполняется `navigate('/projects')`, и возвращается список.

## 8. Исправленные проблемы (центрирование и дублирование сайдбара)

### Проблема
- Двойной сайдбар на странице «Расчёты» (рендерился и в `App.tsx`, и в `CalculationsPage.tsx`).
- Пустое состояние «Начните работу» не центрировалось по вертикали, находилось внизу.
- Калькуляторы «уезжали» вниз при открытии.
- Двойная прокрутка страницы.

### Причина
- Сайдбар был добавлен глобально в `App.tsx` и повторно внутри `CalculationsPage.tsx`.
- Родительские контейнеры (`html, body, #root, .app, .app-layout`) не имели `height: 100%`, а `.main-content` не был flex-контейнером с `justify-content: center`.
- Стили центрирования применялись также к калькуляторам, что приводило к их смещению.

### Решение
1. Удалён `<Sidebar />` из `App.tsx` (оставлен только в `CalculationsPage.tsx`).
2. В `App.css` добавлено:
   ```css
   html, body, #root { height: 100%; margin: 0; padding: 0; }
   .app { min-height: 100%; display: flex; flex-direction: column; }
   .app-layout { display: flex; flex: 1; min-height: 0; }
   .main-content { flex: 1; padding: 20px; overflow-y: auto; display: flex; flex-direction: column; justify-content: center; align-items: center; }
   В global.css добавлен класс .calculations-layout для страницы расчётов:

css
.calculations-layout { display: flex; flex: 1; min-height: calc(100vh - 64px); }
.calculations-layout .main-content { flex: 1; padding: 20px; overflow-y: auto; display: flex; flex-direction: column; justify-content: center; align-items: center; }
Стили центрирования ограничены только пустым состоянием, для режимов calculator, single, all используется обычный поток.

Результат
Сайдбар один, отображается только на странице расчётов.

Пустое состояние центрировано по вертикали и горизонтали.

Калькуляторы (LED, звук, ВКС, эргономика, питание) открываются и не уезжают.

Нет двойной прокрутки.
