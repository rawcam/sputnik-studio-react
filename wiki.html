📌 О проекте

Sputnik Studio — веб-приложение для проектирования AV-систем. Оно позволяет инженерам и проектировщикам:

  • Создавать и вести проекты (бюджет, статусы, дорожные карты).
  • Формировать спецификации оборудования с привязкой к проектам.
  • Рассчитывать AV‑тракты (задержки, битрейт, PoE, мощность).
  • Выполнять специализированные расчёты: LED‑экраны, звуковое давление (SPL), ВКС, эргономика, питание.
  • Управлять финансами проектов (маржинальность, кассовые разрывы).
  • Планировать сервисные работы и регламентное обслуживание.
  • Генерировать техническую документацию на основе шаблонов.

Версия v8 — полная перезапись на React с TypeScript, использующая современные подходы: компонентная архитектура, Redux Toolkit, роутинг, хранилище IndexedDB и локальное хранилище для спецификаций.

🧩 Ключевой функционал

  • Проекты – CRUD, категории, статусы, дорожная карта, приоритеты.
  • Спецификации – создание, копирование, привязка к проекту, фильтрация, сортировка, поиск, отображение в виде сетки/списка.
  • Таблица спецификации – разделы, строки, редактирование полей (вендор, артикул, наименование, количество, цена, скидка, валюта), авторасчёт суммы скидки и итоговой суммы, drag‑and‑drop строк, ресайз колонок, фильтрация по вендору и артикулу, клавиатурная навигация.
  • AV‑тракты – построение цепочек устройств (источники, матрицы, приёмники), расчёт задержек, битрейта, мощности, портов коммутаторов, PoE‑бюджета.
  • Калькуляторы – видео (разрешение, субдискретизация, битрейт), сеть (среда, multicast, QoS), LED (по кабинетам / разрешению), звук (SPL, падение, мощность, RT60), ВКС, эргономика ЭКП, питание (ИБП, охлаждение).
  • Финансовая аналитика – выручка, маржа, рентабельность, кассовые разрывы (по проекту и компании), общехозяйственные расходы (только для директора).
  • Ролевая модель – директор, ГИП, инженер, проектировщик, логист. Разные права доступа к разделам и виджетам.
  • Дашборд – настраиваемые виджеты (финансы, сервис, загрузка сотрудников, риски), карусель проектов, компактный режим.
  • Экспорт/импорт – JSON (полный бэкап) и Excel (выгрузка спецификации).
  • Тёмная тема – поддержка светлой и тёмной темы, переключение через Topbar.

🏗 Архитектура

Проект построен по модульному принципу. Основные директории:

src/
├── assets/          # статика (шрифты, иконки)
├── components/      # переиспользуемые UI‑компоненты
│   ├── common/      # кнопки, модалки, виджет конфигурации
│   ├── layout/      # Topbar, Sidebar, SortableSidebar
│   ├── projects/    # карточки, списки, модалки проектов
│   └── widgets/     # виджеты дашборда
├── db/              # инициализация IndexedDB (Dexie)
├── features/        # фичи (проекты, тракты, калькуляторы, спецификации)
│   ├── projects/    # детальный вид проекта
│   ├── tracts/      # AV‑тракты, устройства
│   ├── video/       # настройки видео и расчёт битрейта
│   ├── network/     # настройки сети, статистика
│   ├── led/, sound/, vc/, ergo/, power/  # калькуляторы
│   └── finances/    # финансовая аналитика
├── hooks/           # кастомные хуки (useAuth, useFinance, useProjectsDb, useServiceVisits)
├── pages/           # страницы (Dashboard, Projects, Calculations, Specifications, Specification)
├── store/           # Redux слайсы (projects, specifications, currency, widgets, tracts, и т.д.)
├── styles/          # глобальные стили, CSS‑переменные
├── types/           # TypeScript типы
├── utils/           # утилиты (работа с хранилищем, финансовые расчёты, modelDB)
├── App.tsx
└── main.tsx

Роутинг — HashRouter (для корректной работы на GitHub Pages).
Состояние — Redux Toolkit, синхронизация с localStorage/IndexedDB.
Спецификации используют глобальные курсы валют (USD/EUR), хранящиеся в отдельном слайсе и localStorage.

👥 Роли пользователей

  • director — полный доступ, видит общехозяйственные расходы и финансовую отчётность компании.
  • pm (руководитель проектов) — видит все проекты, финансовые показатели проектов, но не общехозяйственные расходы.
  • engineer — технические детали, тракты, калькуляторы, загрузку по себе. Финансы скрыты.
  • designer — аналогично инженеру, может редактировать тракты и схемы.
  • logist — видит закупки, сроки поставок, может влиять на расходную часть.

Авторизация локальная: роль хранится в localStorage, в Topbar есть переключатель ролей для тестирования. Компоненты проверяют роль и скрывают/показывают элементы.

📁 Управление проектами

Каждый проект имеет уникальный четырёхзначный идентификатор (shortId), генерируемый в зависимости от категории:

  • new (0000–1999)
  • modernization (2000–3999)
  • service (4000–5999)
  • standard (6000–7999)
  • pilot (8000–9999)

Проекты отображаются в виде карточек или списка с фильтрацией по статусу, приоритету, сортировкой по бюджету, марже, дате, прогрессу.

Детальная страница проекта содержит вкладки:

  • Общая информация – редактируемые поля (название, категория, статус, бюджет, инженер, РП, дата, прогресс).
  • Финансы – доходы и расходы (динамические списки), плановая/фактическая маржа, рентабельность, график денежных потоков.
  • Сервис – список сервисных выездов (добавление, редактирование, удаление).
  • Дорожная карта – план/факт по этапам.
  • Спецификации – отображение спецификаций, привязанных к проекту, с возможностью открыть, открепить, удалить, дублировать, а также создать новую спецификацию для проекта.

📄 Управление спецификациями

Спецификации – это таблицы с оборудованием, которые могут существовать независимо или быть привязаны к проекту. Страница списка спецификаций (/specifications) позволяет:

  • Просматривать спецификации в виде сетки или списка.
  • Искать по названию.
  • Фильтровать по проекту (все проекты / без проекта / конкретный проект).
  • Сортировать по дате, названию, проекту, сумме, количеству позиций.
  • Создавать новую спецификацию (модальное окно с выбором названия и привязкой к проекту).
  • Дублировать, удалять, откреплять от проекта.
  • Переходить на детальную страницу спецификации.

Детальная страница спецификации (/specification/:id) содержит:

  • Редактируемое название и выбор проекта.
  • Курсы валют (глобальные, синхронизированные между всеми спецификациями).
  • Фильтрацию строк по вендору и артикулу.
  • Таблицу с разделами (аккордеоны) и строками данных.
  • Возможность добавлять, удалять, дублировать строки и разделы.
  • Drag‑and‑drop строк (SortableJS).
  • Изменение ширины столбцов (resize).
  • Автоматический пересчёт скидки, цены после скидки, общей суммы (руб).
  • Клавиатурную навигацию (стрелки, Tab, Shift+Tab) с выделением нулевых значений.
  • Информеры (валютная сумма, общая сумма, маржинальность, разбивка по валютам).
  • Экспорт в Excel (все строки или выбранные).
  • Сброс к демо‑данным (локально).

Все изменения спецификации сохраняются автоматически в Redux и localStorage.

🔧 AV‑тракты и расчёты

Страница /calculations содержит сайдбар с настройками и основную область для отображения тракта или калькулятора. Функционал:

  • Создание нескольких независимых трактов.
  • Добавление устройств из базы моделей (источники, передатчики, матрицы, приёмники, дисплеи, коммутаторы, LED‑процессоры).
  • Автоматический расчёт суммарной задержки, общей мощности, битрейта, использования портов коммутаторов, PoE‑бюджета.
  • Редактирование устройств (мощность, PoE, USB, Ethernet).
  • Статистика сети (битрейт, загрузка, порты, multicast, QoS).
  • Статистика питания (PoE, общая мощность, тепловыделение).
  • Экспорт/импорт JSON, сохранение/загрузка из браузера, сброс проекта, печать, Wiki.

🧮 Калькуляторы

  • Видео – разрешение (1080p/4K/8K), субдискретизация (4:4:4/4:2:2/4:2:0), FPS, цветовое пространство, битность → расчёт битрейта.
  • Сеть – среда передачи (Cat5e–Cat8, оптика, беспровод), Multicast, QoS, тип сети, синхронизация, резервирование, статистика.
  • LED – по кабинетам или по разрешению, расчёт размеров, площади, мощности.
  • Звук – SPL на расстоянии, падение SPL от расстояния, изменение SPL от мощности, RT60, подбор громкоговорителей.
  • ВКС – расчёт битрейта кодеков (TrueConf, WebRTC, H.264, H.265), нагрузка на MCU.
  • Эргономика ЭКП – углы обзора, высота установки, расстояние до экрана.
  • Питание – расчёт ИБП (автономность, мощность), кондиционирование (тепловыделение).

💰 Финансовая модель

  • Проект: доходная часть (contractAmount, incomeSchedule), расходная часть (expenseSchedule), фактические доходы/расходы.
  • Расчёт маржи и рентабельности (плановая и фактическая).
  • Кассовый разрыв – накопленный денежный поток по проекту и компании (с учётом общехозяйственных расходов).
  • Общехозяйственные расходы – отдельный слайс, доступен только директору.

🛠 Сервисные работы

В каждом проекте можно создать список сервисных выездов / регламентных работ с полями: дата, тип работ, статус, ответственный, стоимость. На дашборде виджет «Сервис и регламент» отображает ближайшие выезды с учётом роли.

📊 Виджеты дашборда

Дашборд состоит из сетки виджетов (5 колонок). Виджеты можно настраивать через выезжающую панель (видимость, компактный режим). Роли влияют на доступные виджеты:

  • Финансы компании (директор) – выручка, маржа, рентабельность, кассовый разрыв.
  • Финансы проектов (директор, ГИП) – топ‑3 по марже, проекты с отрицательной маржой.
  • Сервис и регламент – ближайшие выезды (фильтрация по роли).
  • Загрузка сотрудников – средняя загрузка инженеров и РП, топ‑3 загруженных.
  • Риски – финансовые, технические, логистические риски (по ролям).
  • Активные проекты – карусель или список проектов с прогрессом.

🎨 UI/UX особенности

  • Тёмная и светлая темы – переключение через кнопку в Topbar, CSS-переменные.
  • Стеклянные карточки (backdrop‑filter, blur).
  • Адаптивная вёрстка – сетка виджетов адаптируется под ширину экрана, карусель проектов с кнопками навигации.
  • Клавиатурная навигация – стрелки для перемещения по ячейкам спецификации, Tab / Shift+Tab.
  • Выделение нулевых значений при фокусе (автоматический .select()) для быстрого ввода.
  • Drag‑and‑drop строк в спецификации (SortableJS).
  • Ресайз колонок с сохранением ширины в localStorage.
  • Информеры с подсказками (атрибут title).

🚀 Установка и запуск

Требования: Node.js 18+ и npm.
  1. Клонировать репозиторий: git clone https://github.com/rawcam/sputnik-studio-react.git
  2. Установить зависимости: npm install
  3. Запустить сервер разработки: npm run dev (откроется http://localhost:3000)
  4. Сборка для продакшена: npm run build (результат в папке dist/)

🌐 Развёртывание на GitHub Pages

Проект автоматически деплоится на GitHub Pages при коммите в ветку main с помощью GitHub Actions (файл .github/workflows/deploy.yml). Адрес: https://rawcam.github.io/sputnik-studio-react/. Для корректной работы роутинга используется HashRouter.

🔮 Планы развития

  • Облачная синхронизация – интеграция с российскими BaaS (Scorocode, Amvera Cloud) или облачными базами данных (VK Cloud, SberCloud) для синхронизации проектов и спецификаций между устройствами.
  • Полноценная авторизация – вход по email/паролю, поддержка ролей через облачный сервис.
  • Интеграция с 1С и внешними API – обмен спецификациями и закупками.
  • Графический редактор блок-схем – визуальное построение AV‑трактов (React Flow).
  • Мобильная версия – адаптация интерфейса для планшетов и смартфонов.
  • Печатные формы – полноценная генерация документов (Word, PDF).
  • Улучшение калькуляторов – добавление новых расчётов (задержки аудио, расчёт кабельных трасс).
  • Расширение ролевой модели – более тонкая настройка прав доступа.

Модульная архитектура на React и TypeScript, использование современных инструментов делают проект удобным для дальнейшего развития и поддержки.
